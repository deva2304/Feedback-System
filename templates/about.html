Both streams are fused into a single purchase probability prediction.

📐 Stream 1: Behavioral CCLF Formulation

The TA-CCLF model factorizes the correlation-constrained behavioral matrix into low-rank latent factors using SVD, with Tikhonov Regularization and Hu-Koren-Volinsky Implicit Confidence Weighting.

// Step 1: Feature extraction
xi = [view_count, session_dur, price, recency, velocity, cart, cat_affinity, rel_interest] ∈ ℝ8

// Step 2: Time-Decayed Correlation Matrix & Tikhonov Regularization
Creg = X̃T · W · X̃ / (n - 1) + λI

// Step 3: SVD decomposition to k={{ model_meta.k_latent }} latent factors
Creg = U · Σ · VT

// Step 4: Correlation-constrained scoring
sbehavioral = Σk pu,k · σk · qi,k · wk
Pbehavioral = σ((s - μ) / τ)
🧬 Stream 2: Multi-Dimensional Semantic Alignment (MDSA)

The MDSA stream encodes product names into dense 384-dimensional vectors using a pretrained transformer model (all-MiniLM-L6-v2) and computes semantic alignment with the user's browsing history.

// Step 1: Semantic embedding via pretrained transformer
ei = Encoder(product_namei) ∈ ℝ384

// Step 2: Time-decayed cosine similarity with browsing history
sim(ei, ej) = (ei · ej) / (||ei|| × ||ej||)
wj = e-Δtj / λsem (Ebbinghaus time decay)

// Step 3: Weighted mean semantic alignment
Salign = Σj (wj × sim(ei, ej)) / Σj wj

// Step 4: Multi-dimensional decomposition
D₁ = Functional alignment (overall semantic similarity)
D₂ = Brand/Quality tier (lexical overlap clustering)
D₃ = Category coherence (same product family ratio)
⚖️ Hybrid Fusion Layer

The final purchase probability is computed by fusing both streams with adaptive weights. Cold-start products (first view) receive higher semantic weight since there is no behavioral data yet.

// Normal mode (views > 1)
Pfinal = 0.65 × Pbehavioral + 0.35 × Salign

// Cold-start mode (first view)
Pfinal = 0.45 × Pbehavioral + 0.55 × Salign

Cold Start Advantage: New products with zero behavioral history still get meaningful predictions via semantic alignment with the user's browsing profile.

Engagement Amplification: As the user browses more, behavioral weight increases and the model grows more confident in its predictions.

⚡ Hybrid Real-Time Prediction Pipeline
1
Extension Captures
Browser extension detects Amazon product pages and sends product name, price, and URL to the local Flask server.
2
Behavioral Feature Extraction
8 behavioral features are extracted from the session: view count, duration, price, recency, velocity, cart status, category affinity, and relative interest.
3
Semantic Embedding (MDSA)
Product name is encoded into a 384-dimensional dense vector via all-MiniLM-L6-v2 transformer. Result is cached for instant reuse.
4
Correlation Matrix + SVD
Pearson correlation matrix is built across products, regularized (Tikhonov), and decomposed into k={{ model_meta.k_latent }} latent factors via SVD.
5
Multi-Dimensional Semantic Alignment
Cosine similarity between current product embedding and time-weighted history embeddings, decomposed across 3 dimensions: Functional, Brand/Quality, Category.
6
Behavioral SVD Scoring
Correlation-constrained dot product with Hu-Koren-Volinsky implicit confidence weighting, calibrated via Platt scaling sigmoid.
7
Hybrid Fusion
Behavioral (65%) and semantic (35%) scores are fused. Cold-start products shift to 45%/55% to compensate for missing behavioral data.
🎯 Why Correlation Constraints?

In standard latent factor models, correlated features can dominate predictions. The correlation constraint applies inverse-sqrt weighting from SVD singular values, penalizing redundant signals.

Independent features receive higher weight — they provide unique signal.

Redundant features receive lower weight — the model deduplicates automatically.

🧬 Why Multi-Dimensional Semantic Alignment?

Behavioral models alone cannot understand what a product is — only how the user interacts with it. MDSA solves this by adding textual understanding:

Cold Start Solution: A brand new product with zero views still gets a meaningful score based on its semantic similarity to previously browsed items.

Cross-Category Discovery: MDSA knows that "AirPods" and "Sony WH-1000XM5" are semantically related even though their categories might differ.

3 Alignment Dimensions: Functional purpose, Brand/Quality tier, and Category coherence provide granular understanding of product similarity.

🔒 Privacy Architecture

All data stays on your local machine. The extension communicates exclusively with your local Flask server at 127.0.0.1:5000. No browsing data ever leaves your computer. All SVD computation, correlation analysis, and prediction happen locally in real-time.
