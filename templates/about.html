📐 Mathematical Formulation

The TA-CCLF model factorizes the correlation-constrained behavioral matrix into low-rank latent factors using Singular Value Decomposition, upgraded with Tikhonov Regularization and Hu-Koren-Volinsky Implicit Confidence Weighting.

// Step 1: Feature extraction
xi = [view_count, session_dur, price, recency, velocity, cart, cat_affinity, rel_interest] ∈ ℝ8

// Step 2: Time-Decayed Correlation Matrix (Ebbinghaus) & Tikhonov Regularization
W = diag(e-t / λdecay) (exponential time decay for novelty scaling)
Creg = X̃T · W · X̃ / (n - 1) + λI where X̃ = normalized(X)

// Step 3: SVD decomposition
Creg = U · Σ · VT
Truncate to k={{ model_meta.k_latent }} latent factors

// Step 4: Latent space projection (Hu, Koren, Volinsky Confidence)
cui = 1 + α(log(1 + views)) + β(cart) + γ(affinity)
qi = UkT · xi (item latent)
pu = Σ (cuj · UkT · xj) / Σ cuj (user latent, implicitly weighted)

// Step 5: Correlation constraint
wk = 1 / √σk (penalize redundant correlated features)

// Step 6: Score computation
s = Σk pu,k · σk · qi,k · wk

// Step 7: Platt Scaling Probability Calibration
P(y=1) = σ((s - μ) / τ) (temperature-calibrated logistic function)

⚡ Real-Time Prediction Pipeline
1
Extension Captures
Browser extension detects Amazon product pages and sends product name, price, and URL to the local Flask server. Every page load is tracked — revisits increment the view count.
2
Feature Extraction
8 behavioral features are extracted from the session: view count, duration, price, recency, velocity, cart status, category affinity, and relative interest.
3
Correlation Matrix
A Pearson correlation matrix is computed across all tracked products to identify which features provide independent signal vs. redundant information.
4
SVD Decomposition
The correlation matrix is decomposed via SVD into k={{ model_meta.k_latent }} latent factors. These capture the dominant patterns of correlated behavior.
5
Constrained Scoring
User and item latent vectors are computed, then a correlation-constrained dot product produces the raw prediction score. Redundant features are penalized.
6
Sigmoid Calibration
The raw score is passed through a calibrated sigmoid function to produce a purchase probability between 0% and 100%, with realistic bounds.
🎯 Why Correlation Constraints?

In standard latent factor models, highly correlated features (e.g., view count and session duration) can dominate predictions, leading to redundant signals and lower accuracy.

The correlation constraint applies an inverse-sqrt weighting from the SVD singular values. This means:

Independent features (low correlation with others) receive higher weight — they provide unique signal.

Redundant features (high correlation with others) receive lower weight — the model deduplicates the signal automatically.

This constraint is especially powerful for this use case because behavioral signals like "view count" and "view velocity" are naturally correlated — CCLF ensures the model doesn't double-count them.

🔒 Privacy Architecture

All data stays on your local machine. The extension communicates exclusively with your local Flask server at 127.0.0.1:5000. No browsing data ever leaves your computer. All SVD computation, correlation analysis, and prediction happen locally in real-time.
